# 集中测试清单

测试日期：待执行

测试目标：确认登录、绑定、情侣数据隔离、核心页面读写共享、错误状态提示都能在微信开发者工具和真机中稳定运行。

## 1. 测试前准备

### 1.1 云函数上传检查

- 确认已上传并部署 `cloudfunctions/login`。
- 确认已上传并部署 `cloudfunctions/couple`。
- 在云开发控制台分别打开两个云函数日志，确认最近一次调用没有 `INTERNAL_ERROR`、`NO_OPENID`、`UNKNOWN_ACTION`。

### 1.2 数据库集合检查

需要存在这些集合：

| 集合名 | 用途 |
|---|---|
| `users` | 当前用户初始化、保存 `openid/currentCoupleId` |
| `couples` | 情侣关系主表 |
| `invitations` | 邀请绑定记录 |
| `remember_couples` | “我们页”头像和昵称兼容表 |
| `love_wishes` | 心愿清单 |
| `love_album` | 甜蜜相册 |
| `love_dates` | 约会记录 |
| `love_memorials` | 纪念日 |
| `love_goals` | 仪表盘共同目标 |
| `heartbeat_moments` | 心动瞬间 |
| `heartbeat_notes` | 小纸条 |
| `heartbeat_bank` | 心动金库 |
| `heartbeat_task_logs` | 每日任务打卡 |

### 1.3 数据库权限检查

- 联调阶段可以临时允许登录用户读写，便于确认功能链路。
- 正式使用前不要长期保持宽松权限。
- `users/couples/invitations` 建议最终仅允许云函数写。
- 所有业务集合至少要保证只能读写当前用户所属 `coupleId` 下的数据。
- 如果出现 `permission denied`，先截图控制台错误，再记录对应集合和页面。

### 1.4 建议索引检查

- `love_wishes`: `coupleId + status + priority + createdAt`
- `love_album`: `coupleId + createdAt`
- `love_dates`: `coupleId + dateTs`
- `love_memorials`: `coupleId + updatedAt`
- `love_goals`: `coupleId + updatedAt`
- `heartbeat_moments`: `coupleId + createdAt`
- `heartbeat_notes`: `coupleId + toOpenid + read + createdAt`
- `heartbeat_bank`: `coupleId`
- `heartbeat_task_logs`: `coupleId + openid + date + taskId`

## 2. 单人未绑定状态测试

1. 清理开发者工具缓存，或换一个未绑定微信账号测试。
2. 打开小程序首页，确认能正常启动。
3. 进入“我们”页，确认显示未绑定或等待绑定状态。
4. 逐个打开以下页面，确认不报红色错误，并显示“请先绑定情侣关系”：
   - 心愿清单 `pkg/wishlist/wishlist`
   - 甜蜜相册 `pkg/photoalbum/photoalbum`
   - 约会记录 `pkg/daterecord/daterecord`
   - 纪念日 `pkg/memorial/memorial`
   - 仪表盘 `pkg/timeline/timeline`
   - 心动页 `pkg/heartbeat/heartbeat`
5. 未绑定状态下点击新增、上传、打卡、发纸条等按钮，应该只提示先绑定，不应该写入业务集合。
6. 如果已经生成邀请但对方还没有接受，业务页面应显示等待绑定提示，不应该允许新增共享数据。

## 3. 生成邀请测试

1. 用你的微信进入“我们”页。
2. 点击生成邀请或分享入口。
3. 预期结果：
   - `users` 中你的文档存在，`currentCoupleId` 不为空；
   - `couples` 中存在对应文档，`members` 至少包含你的 `openid`；
   - `invitations` 中存在 `status: pending` 的邀请；
   - 页面显示等待 Ta 绑定；
   - 分享路径包含 `inviteId` 和 `token`。

## 4. 双人绑定测试

后续需要你女朋友参与时执行：

1. 你先生成邀请并分享给她。
2. 她从分享卡片进入小程序。
3. 她进入后预期显示绑定成功。
4. 两台设备都重新进入“我们”页。
5. 预期结果：
   - `users` 中两个人的 `currentCoupleId` 相同；
   - `couples.members` 有两个不同 `openid`；
   - `couples.status` 为 `active`；
   - `invitations.status` 为 `accepted`；
   - `remember_couples` 中两个人 openid 至少能对应上。

## 5. 核心页面读写共享测试

每个页面都按这个通用方法测：

1. A 设备新增一条数据。
2. A 设备下拉刷新或重新进入页面，确认能看到。
3. B 设备进入同一页面，确认也能看到。
4. B 设备修改、完成、收藏、置顶或删除。
5. A 设备刷新，确认变化同步。
6. 打开数据库对应集合，确认每条记录都有正确 `coupleId`。
7. 确认没有写入真实 openid 硬编码，涉及用户身份的字段应来自当前登录态。

## 6. 每个页面的测试步骤

### 6.1 心愿清单 `love_wishes`

- 新增一个心愿，填写标题、备注、优先级，可选照片。
- 确认记录包含 `coupleId/status/createdAt/photoFileID`。
- 点击完成，确认 `status` 变为 `done`，`doneAt` 存在。
- 点击撤销，确认 `status` 变回 `todo`。
- 删除记录，确认页面消失。
- 空列表时应显示空状态。

### 6.2 甜蜜相册 `love_album`

- 上传 1 至 3 张照片。
- 确认记录包含 `coupleId/fileIDs/coverRatio/createdAt`。
- 搜索标题或标签。
- 收藏和取消收藏。
- 编辑标题、标签、描述。
- 删除相册记录。
- 空相册时应显示空状态。

### 6.3 约会记录 `love_dates`

- 新增约会，填写标题、日期、地点、花费、心情、备注，可选照片。
- 确认记录包含 `coupleId/date/dateTs/photoFileIDs`。
- 切换时间轴和日历。
- 点击照片预览，确认不会报错。
- 编辑记录。
- 生成海报，确认能生成或有失败提示。
- 删除记录。

### 6.4 纪念日 `love_memorials`

- 新增纪念日，选择日期、类型、颜色、是否每年重复。
- 确认记录包含 `coupleId/date/type/color/repeat/pinned`。
- 切换“全部/即将到来”。
- 置顶和取消置顶。
- 编辑记录。
- 删除记录。

### 6.5 仪表盘与共同目标 `love_goals`

- 打开仪表盘，确认统计卡片能加载。
- 点击盲盒“加入共同目标”，确认 `love_goals` 新增记录并带 `coupleId`。
- 手动新增共同目标。
- 勾选目标完成状态。
- 删除目标。
- 如果业务集合为空，今日待办和统计不应报错。

### 6.6 心动瞬间 `heartbeat_moments`

- 新增心动瞬间，填写标题、类型、心动值、文字，可选照片或语音。
- 确认记录包含 `coupleId/openid/date/type/title`。
- 收藏和取消收藏。
- 删除瞬间。
- 空列表时应显示空状态。

### 6.7 小纸条 `heartbeat_notes`

- 发送文字纸条。
- 发送图片纸条。
- 发送语音纸条。
- 如果能识别到对方，测试私密纸条；未识别到对方时，应看到提示。
- 打开纸条详情，确认未读状态变化。
- 确认记录包含 `coupleId/fromOpenid/toOpenid/noteType/read`。

### 6.8 心动金库 `heartbeat_bank`

- 新增心动瞬间后，确认金库分数增加。
- 发送纸条后，确认金库分数增加。
- 完成每日任务后，确认金库分数增加。
- 确认 `heartbeat_bank` 记录包含 `coupleId/points`。

### 6.9 任务日志 `heartbeat_task_logs`

- 打开心动页任务 Tab。
- 点击“完成打卡”。
- 确认 `heartbeat_task_logs` 新增记录，包含 `coupleId/openid/date/taskId`。
- 再次点击不应重复打卡。
- 换一个任务后再次测试，确认任务状态按当前任务刷新。

## 7. 云函数失败和数据库失败测试

可以临时模拟：

- 删除或改名 `couple` 云函数后打开页面，应显示绑定状态读取失败。
- 临时收紧某个业务集合权限，应显示对应页面加载失败。
- 断网或弱网时，页面不应红屏，应有 toast 或错误态。

测试后记得恢复云函数和数据库权限。

## 8. 测试失败时请提供的信息

请把以下信息发给我继续排查：

- 失败页面截图。
- 微信开发者工具 Console 完整报错。
- Network 或云调用面板中失败的云函数名、入参、返回值。
- 云开发控制台中 `login` 和 `couple` 云函数日志。
- 相关数据库集合中的记录截图或字段摘录：
  - `users.openid/currentCoupleId`
  - `couples._id/members/status`
  - `invitations._id/status/coupleId/inviterOpenid/inviteeOpenid`
  - 失败业务集合记录的 `_id/coupleId`
- 如果是上传图片或语音失败，提供云存储错误信息和 `cloudPath`。
- 如果是权限失败，提供错误码、错误信息、集合名和当时所在页面。
