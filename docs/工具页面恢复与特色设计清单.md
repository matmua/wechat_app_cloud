# 工具页面恢复与特色设计清单

## 1. 本次结论

本次重点从“统一模板”退回到“工具各有玩法”。已恢复小程序名称为“爱木长诗”，并根据 git 历史和当前入口文案，恢复了几个原有趣味工具的定位：`美食猜猜`、`树洞袜子`、`呼叫对方`、`小小计划`、`大大喜欢`、`打发时间`、`AI助手`。其中 `美食猜猜` 在历史中只找到入口和空模板，没有找到完整旧实现，因此本次在原路由上重做了一个最小可用的随机美食抽卡工具。

本次没有修改 `login/couple` 云函数，也没有改动心愿、相册、约会、纪念日、心动瞬间等已经接入云数据的核心业务逻辑。

## 3. 哪些工具是恢复的

- 恢复品牌名：`爱木长诗`。
- 恢复原工具页趣味入口：`美食猜猜`、`树洞袜子`、`呼叫对方`、`小小计划`、`大大喜欢`、`打发时间`、`AI助手`。
- 恢复工具页彩色入口氛围，不再让工具页看起来像普通管理后台。

## 4. 哪些工具是重新设计的

- `美食猜猜`：没有找到完整旧实现，本次按原用途重做。
- `树洞袜子`、`呼叫对方`、`打发时间`、`小小计划`、`大大喜欢`：历史里都是空模板或路由错配，本次按原入口名称重做最小可用玩法。
- `AI助手`：保留 AI 助手定位，当前为“小诗助手”，真实模型接口待后续通过云函数配置。
- `恋爱目标`：本次从统一占位改成带进度条的本地目标页，使用 `coupleId` 做本地隔离。

## 5. 仍然只是占位或后续再精修

`weather`、`datePlan`、`giftSuggest`、`memorialDay`、`photolove`、`coupleTasks`、`loveCalendar`、`relationshipAdvice`、`loveStory`、`surpriseIdeas`、`emotionAnalysis`、`moreServices` 仍然不同程度保留占位性质。后续建议逐个明确是否真的需要独立功能，避免继续出现“入口名”和“页面实际功能”不一致的问题。

## 6. 后续重点测试

1. 工具页入口：确认“美食猜猜、树洞袜子、呼叫对方、小小计划、大大喜欢、打发时间、AI助手、恋爱目标”都能打开正确页面。
2. 美食猜猜：新增候选项、删除候选项、连续抽卡、清空历史、恢复默认。
3. 树洞袜子：点击不同袜子、随机抽取、清空抽取记录。
4. 呼叫对方：切换呼叫类型、随机换一个、复制文案。
5. 打发时间：抽挑战、开始计时、重置计时、离开页面后不继续计时。
6. 小小计划：新增便签、完成/重新来、撕掉、本地刷新后是否保留。
7. 大大喜欢：新增偏好、切换分类、删除偏好。
8. AI助手：测试快捷 prompt、发送消息、未部署 `aiChat` 时的“AI 接口待配置”提示。
9. 恋爱目标：未绑定时不能记录；绑定后新增目标、进度 +10%、删除目标，并确认不同 `coupleId` 不共用本地目标。
