# 特色工具语义纠偏与设计清单

## 1. 本次结论

本次优先纠正四个语义偏差最大的工具：`树洞袜子`、`打发时间`、`对方天气`、`经期记录`。它们已经不再共用统一占位模板，而是分别改成隐藏悄悄话抽屉、街机小游戏中心、远方天气卡、私密关心日历。

本次没有修改 `login/couple` 云函数，也没有修改小程序名称“爱木长诗”。涉及情侣共享或敏感数据的页面都先读取当前绑定状态，未绑定时不写入数据；写入时包含 `coupleId`，私密内容包含 `ownerOpenid`。

## 3. 树洞袜子如何隐藏和定时公开

- 新集合：`sock_secrets`。
- 写入字段：`coupleId`、`ownerOpenid`、`contentType`、`text`、`fileIDs`、`visibility`、`revealAt`、`createdAt`、`updatedAt`。
- `visibility = public`：保存后对方可见。
- `visibility = hidden`：只查询和展示给作者自己。
- `visibility = scheduled`：作者自己随时可见；对方只查询 `revealAt <= 当前时间` 的内容。
- 图片、视频、音频会先上传到云存储，数据库只保存 `fileID`，不会保存本地临时路径。

生产环境建议继续补云函数或数据库权限规则，确保非作者不能绕过前端查询隐藏内容。

## 4. 打发时间现在有哪些小游戏

- `贪吃蛇`：支持开始、暂停、重置、分数、吃食物、撞墙/撞自己结束；控制区已改为不超屏的十字方向键，并保留滑动控制。
- `扫雷`：固定小棋盘，支持翻开格子、数字提示、踩雷失败、全部安全格打开胜利、重新开始。
- `打飞机`：支持左右移动、发射子弹、敌机下落、碰撞判断、分数、游戏结束、重新开始。
- `俄罗斯方块`：支持方块下落、左右移动、旋转、加速下落、消行、分数、游戏结束、重新开始。
- 四个小游戏都使用小程序基础 WXML/WXSS/JS 实现，不依赖第三方游戏引擎；当前是最小可玩版本，后续可继续补音效、难度和最高分。

## 5. 天气如何显示自己和对方天气

- 新集合：`partner_locations`。
- 当前用户只能主动同步自己的城市/经纬度/更新时间。
- 页面通过 `weather` 云函数分别读取 `target=self` 和 `target=partner`，查询自己与对方在当前 `coupleId` 下同步的位置。
- 自己未同步时显示“你还没有同步自己的位置”。
- 对方未同步时显示“等待对方同步位置”。
- 默认使用 Open-Meteo，云函数用自己或对方同步的经纬度请求真实天气服务，返回温度、天气、体感、湿度、风速、weather_code 和关心文案。
- Open-Meteo 非商业使用不需要 API key；如果只手动填写城市但没有经纬度，页面会提示先同步位置。
- 如果环境变量 `WEATHER_PROVIDER=qweather`，云函数改用 QWeather；此时 API key 通过 `QWEATHER_API_KEY` 或 `WEATHER_API_KEY` 配置，不写进前端代码。

## 6. 经期记录如何保护隐私

- 新集合：`period_records`。
- 每条记录包含 `coupleId`、`ownerOpenid`、`periodStart`、`periodEnd`、`cycleLength`、`periodLength`、`notes`、`visibility`。
- `private`：仅本人查询。
- `careOnly`：对方只看到“需要多关心”的提示，不展示日期和备注。
- `shared`：对方可看到共享日期和备注。
- 页面顶部有隐私提示，默认保存模式是 `careOnly`，避免默认暴露过多细节。

生产环境建议继续补云函数或数据库字段级权限，因为生理期属于敏感隐私数据。

## 7. 后续需要接云函数或真实 API

- `weather` 云函数：代码已新增，默认 Open-Meteo 不需要 API key；如果切到 QWeather，才需要在微信云开发中配置天气 API key，详见 `docs/天气API接入说明.md`。
- `sock_secrets` 云函数：服务端按 `ownerOpenid/visibility/revealAt` 返回可见内容，避免前端权限过宽。
- `period_records` 云函数：服务端按隐私模式返回摘要或详情，避免 `careOnly` 记录泄露细节。

## 8. 微信开发者工具测试重点

1. 未绑定状态：树洞袜子、对方天气、经期记录都不能写入数据。
2. 树洞袜子：分别保存文字、图片、视频、音频；测试公开、隐藏、定时公开；确认数据库只保存 `fileID`。
3. 打发时间：分别测试贪吃蛇、扫雷、打飞机、俄罗斯方块的开始、暂停、重置、胜负和小屏宽度。
4. 对方天气：自己手动填写城市并同步；另一个账号同步后，当前账号能看到对方城市和天气卡；上传 `weather` 云函数并配置 key 后再测真实天气。
5. 经期记录：保存 private/careOnly/shared 三种记录；确认对方只看到允许展示的信息。
