# 剩余页面设计与实现说明

更新时间：2026-06-18

本轮目标是把原先仍偏空、偏占位的剩余页面推进到“测试版可用”：页面能打开，有明确用途，有基础交互，有自己的视觉特色；绑定后写入都带 `coupleId` 和 `creatorOpenid`；未绑定时阻止情侣共享数据写入。当前仍是测试版，新增页面多数使用前端直连云数据库，并提供本地兜底，正式使用前仍建议按最终验收报告继续收紧敏感权限。

## 5. 新增或使用的集合

请在云开发数据库中创建或检查这些集合：

| 集合 | 用途 | 必要性 |
| --- | --- | --- |
| `date_plans` | 约会路线卡 | 使用约会规划时必须 |
| `gift_ideas` | 礼物灵感和已送出状态 | 使用礼物建议时必须 |
| `memorial_preparations` | 纪念日准备信封 | 使用纪念日准备时必须 |
| `photo_stories` | 照片故事卡，图片保存 `fileIDs` | 使用照片故事时必须 |
| `couple_tasks` | 情侣任务、打卡和进度 | 使用情侣任务时必须 |
| `love_calendar_events` | 自定义日历事件 | 使用恋爱日历新增事件时必须 |
| `relationship_advice_favorites` | 关系建议收藏 | 收藏建议时必须 |
| `surprise_ideas` | 惊喜盲盒 | 使用惊喜灵感时必须 |
| `emotion_records` | 情绪记录和趋势 | 使用情绪温度计时必须 |
| `service_feedback` | 更多服务反馈记录 | 可选；未绑定时会保存本地 |

所有新增情侣共享集合写入时都会带 `coupleId` 和 `creatorOpenid`。当前测试版在集合不存在或权限不足时，多数页面会切到本地缓存并提示“云端暂不可用”，方便先测试页面交互。

## 6. 仍然只是部分完成的点

- `photolove` 已使用云存储上传图片，但没有做云存储文件删除；删除故事时只删除数据库记录。
- `loveCalendar` 当前是前端聚合，某个来源集合无权限时会降级，不保证所有历史数据都完整显示。
- `relationshipAdvice` 是结构化建议和收藏，不是真实 AI；深度生成仍交给 `pkg/moodDiary/moodDiary` 后续接 `aiChat`。
- `emotionAnalysis` 是情绪记录和趋势，不会读取聊天、树洞、经期、定位等隐私，也不做自动 AI 分析。
- `moreServices` 的反馈是轻量测试记录，不等同于正式反馈系统。
- 本轮没有做敏感集合云函数权限迁移，正式使用前仍需要处理权限。

## 7. 微信开发者工具测试步骤

1. 确认小程序能正常编译，工具页可打开。
2. 在“我们页”确认已绑定；未绑定时进入新增页面应看到“请先绑定情侣关系”或收藏受限提示。
3. 在云开发数据库创建上表列出的集合。
4. 进入 `约会规划`：新增路线、增删步骤、勾选准备事项、标记完成、删除。
5. 进入 `礼物建议`：新增礼物、按场景筛选、随机拆盒、置顶、标记已送出、删除。
6. 进入 `纪念日准备`：新增准备信封、勾选准备事项、标记完成、跳转正式纪念日。
7. 进入 `照片故事`：选择图片、保存故事、预览图片、删除故事；如上传失败，检查云存储权限。
8. 进入 `情侣任务`：创建一次性任务和连续打卡任务，打卡到完成，重置和删除。
9. 进入 `恋爱日历`：切换月份、新增自定义事件，查看来自约会规划/今日计划/纪念日/经期记录的事件点。
10. 进入 `关系建议`：切换场景、复制话术、收藏、删除收藏、跳转小诗助手。
11. 进入 `惊喜盲盒`：新增隐藏惊喜，用另一端账号确认只能看到未拆盲盒提示；公开后再查看。
12. 进入 `情绪温度计`：记录心情、标签、分数，查看 7 天趋势。
13. 进入 `功能地图`：保存反馈记录，未绑定和已绑定状态分别测试。

如果失败，请提供：页面截图、控制台报错、对应集合的一条数据库记录、是否显示“本地兜底”提示、当前用户是否已绑定。
