# 趣味化体验与我们页实用化说明

更新日期：2026-06-18

## 1. 本轮目标

本轮重点不是安全权限迁移，也不是统一模板化 UI，而是把“爱木长诗”的日常使用体验做得更有趣、更像两个人的小空间：

- AI 助手继续保留为真正的 AI 助手，不降级、不改名为普通本地工具。
- 工具页从固定菜单改成分组/分页的小工具箱。
- “我们页”下方格子从静态入口改成可新增、可打勾、可保存历史的今日计划，以及今日心情、今日一句话和快捷入口。
- 页面视觉继续保持温柔、轻量、情侣日常感，但避免所有工具长成同一种卡片模板。

本轮没有修改 `login` / `couple` 云函数核心逻辑，没有修改小程序名称“爱木长诗”，没有提交 commit。

## 2. AI 助手如何保留

页面：`pkg/moodDiary/moodDiary`

本轮已把该页面调整为 **小诗助手**，定位仍然是“爱木长诗里的恋爱 AI 小助手”。

当前页面保留了：

- 对话界面；
- 快捷 prompt 入口；
- AI 接入待配置提示；
- `aiChat` 云函数调用结构；
- API key 只放云函数环境变量的说明；
- 长按复制回复；
- 清空对话；
- 发送消息。

快捷功能包括：

| 快捷功能 | 用途 |
| --- | --- |
| 哄 Ta 开心 | 生成自然安慰话 |
| 一句情话 | 生成短句情话 |
| 约会计划 | 生成轻松约会安排 |
| 礼物建议 | 给礼物方向 |
| 树洞润色 | 润色悄悄话 |
| 经期关心 | 给日常关心建议 |
| 今天吃什么 | 解决饭点纠结 |

当前还没有真实 `cloudfunctions/aiChat`，所以页面会提示“AI 接口待配置”。后续接入 DeepSeek / OpenAI 时，建议新增 `cloudfunctions/aiChat`：

- 前端调用：`wx.cloud.callFunction({ name: 'aiChat', data: { action: 'chat', scene, message, messages } })`
- 云函数环境变量：`AI_PROVIDER`、`AI_API_KEY`、`AI_API_HOST`、`AI_MODEL`
- API key 不写入前端；
- 默认不自动发送树洞、经期、定位等敏感内容。

## 3. 工具页分组 / 分页 / 搜索

页面：`pages/matters/matters`

本轮已把原来的固定工具宫格改成“爱木长诗的小工具箱”。

当前分组：

| 分组 | 说明 | 代表工具 |
| --- | --- | --- |
| 今日常用 | 今天最容易用上的工具 | 心愿清单、美食猜猜、天气卡片、小诗助手、今日计划、纪念日 |
| 记录回忆 | 把一起发生的事收好 | 相册、约会记录、心动瞬间、仪表盘 |
| 趣味互动 | 无聊时一起玩一点 | 树洞袜子、打发时间、呼叫对方、小小计划、大大喜欢 |
| 实用工具 | 天气、经期和提醒 | 经期记录、约会规划、礼物建议、恋爱目标 |
| AI 助手 | 后续接入 DeepSeek / OpenAI | 小诗助手、关系建议、情绪分析 |
| 后续功能 | 先留入口，慢慢补齐 | 更多服务、随机约会、未来信箱 |

交互变化：

- 支持顶部分类 tab；
- 支持左右 swiper 切换分组；
- 每组工具再分页展示，每页最多 6 个入口；
- 支持搜索工具名称、说明、关键词；
- 占位功能会标“后续”；
- 已完成核心功能优先出现在“今日常用”。

## 4. 我们页下方格子变成了什么

页面：`pages/remembers/remembers`

原来的下方格子现在改成真实模块：

| 模块 | 当前能力 | 数据 |
| --- | --- | --- |
| 今日计划 | 新增计划、打勾完成、取消完成、删除、完成进度、查看历史 | `daily_plans` |
| 今日心情 | 选择心情、写备注、保存自己的今日状态、显示双方心情 | `daily_moods` |
| 今日一句话 | 和今日心情一起保存，显示最近一句 | `daily_moods.dailyWord` |
| 快捷入口 | 天气、纪念日、树洞、约会、美食、小诗助手等可点击跳转 | 不单独存储 |

未绑定时：

- 页面不会写入 `daily_plans` 或 `daily_moods`；
- 会提示先绑定情侣关系；
- 邀请码绑定流程仍保留。

已绑定时：

- 今日计划按 `coupleId + date` 保存；
- 今日心情按 `coupleId + date + creatorOpenid` 保存；
- 双方可以看到同一个情侣空间当天的计划和心情。

## 5. 今日计划如何保存历史

集合：`daily_plans`

推荐字段：

| 字段 | 说明 |
| --- | --- |
| `coupleId` | 当前情侣空间 |
| `date` | 日期，格式 `YYYY-MM-DD` |
| `title` | 计划标题 |
| `done` | 是否完成 |
| `creatorOpenid` | 创建者 |
| `createdAt` | 创建时间 |
| `updatedAt` | 更新时间 |
| `doneAt` | 完成时间 |

页面能力：

- 新增当天计划；
- 点击圆圈打勾；
- 再点取消完成；
- 删除计划；
- 显示“今天完成了几件小事”；
- 查看最近 7 天左右的历史分组；
- 历史里显示当天计划数、完成数和未完成项。
- 云数据库集合不存在或权限未开放时，会先保存到本地缓存，并在页面显示本地模式提示；创建集合后仍优先使用云端。

这不是复杂 todo 软件，定位是情侣日常的小计划板。

## 8. 需要创建或检查的集合

本轮新增使用：

| 集合 | 是否必须 | 用途 |
| --- | --- | --- |
| `daily_plans` | 是，如果使用今日计划 | 我们页今日计划和历史 |
| `daily_moods` | 是，如果使用今日心情/一句话 | 双方今日心情和今日一句话 |

建议索引：

- `daily_moods`: `coupleId + date + creatorOpenid`

当前仍是测试版，前端直接写这两个集合可以接受；正式发布前再按整体安全计划迁移。

## 9. 微信开发者工具测试步骤

1. 进入“工具页”，确认顶部分类可以点击。
2. 左右滑动工具页分组，确认不会横向溢出。
3. 搜索“天气”“AI”“树洞”“吃什么”，确认能筛选出对应工具。
4. 点击“后续”工具，确认能打开占位页，不误以为已完成。
5. 进入“小诗助手”，确认页面名称不是本地心情助手。
6. 点击快捷 prompt，确认输入框填入对应内容。
7. 点击发送；如果还没部署 `aiChat`，应显示“AI 接口待配置”的温柔提示。
8. 进入“我们页”，绑定状态正常时，新增一条今日计划。
9. 点击计划圆圈，确认能完成；再点一次确认能取消完成。
10. 删除计划，确认页面刷新。
11. 点击“查看历史”，确认按日期分组显示完成数。
12. 选择今日心情，填写备注和今日一句话，点击保存。
13. 如果提示“已先保存到本地”，说明 `daily_plans` 或 `daily_moods` 集合/权限还没准备好；创建集合后再测云端共享。
14. 使用另一个账号打开“我们页”，确认能看到同一个 `coupleId` 下的计划和对方心情。
